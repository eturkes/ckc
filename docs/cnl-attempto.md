# Attempto ecosystem — CNL family report for CKC

Provenance: `cnl-ir-research` workflow harvest (Claude subagents, 2026-07-07); URLs/licenses verified at fetch time. Companions: `cnl-multilingual-ja.md`, `cnl-landscape.md`, `cnl-design-codex.md`. Consumer: SPEC §10 (ClinicalCNL).

## 1. System inventory

### ACE (Attempto Controlled English), language spec 6.7
- What: general-purpose CNL = unambiguous FOL fragment wearing full-English syntax. Dev at Univ. Zurich 1995–~2013 (Fuchs, Kaljurand, Kuhn, Schwertel, Höfler). Spec frozen at 6.7 (2013); reference docs = Construction Rules + Interpretation Rules + Syntax Report + DRS Report 6.6.
- Semantics target: DRS (Discourse Representation Structure) = notational FOL variant. DRS ops: `=>` (if-then), `-` (neg), `~` (NAF), `v` (disj), `can/1` `must/1` `should/1` `may/1` (modal boxes), `:` (sub-DRS label). Crucially, DRS report states should/may have **no fixed semantics — domain decides** (deliberate underspecification hook).
- Construct inventory (6.7): sg/pl countable + mass NPs; determiners `a/every/no/not every/all`; generalized quantifiers `at least/at most/more than/less than/exactly N` (+ distributive `each of`); measurement nouns (`2 m`, `31.2°C`, `3 kg of apples`); integers/reals, arithmetic exprs (+,-,*,/,^, not evaluated), comparison formulas `= \= > >= < =<` (not evaluated); strings, sets `{}`, lists `[]`; variables `X`, `D1` as appositions; proper names (multiword hyphenated `United-Nations`); adjectives incl. comparative/superlative + `as…as`/`more…than`; relative clauses (coordinatable, pronoun must repeat); Saxon + of-genitives; verbs intrans/trans/ditrans, passive requires `by`; hyphenated phrasal/prepositional verbs (`fills-in`, `fond-of`); VP negation `does not`, NAF `does not provably`; **modals: `can` (possibility), `must/has to` (necessity), `should` (recommendation), `may` (admissibility)** + sentence forms `it is (not) possible/necessary/recommended/admissible that`; sentence negation `it is false that`; NAF sentence `it is not provable that`; if-then; sentence subordination (`believes that`); and/or with fixed precedence; there-is; yes/no + wh + how-many queries; commands (`John, go!`). Tense: simple present only — **no temporal operators, no durations**; only `when/where` questions + ordinary PPs (`for 7 days` = an uninterpreted PP).
- Determinism (interpretation rules force one reading): PP always modifies verb never noun; adverb between verbs → binds left; relative clause → immediately preceding noun; `and` > `or`, comma-variants `,and`/`,or` invert; quantifier scope = textual order, extends to sentence end (no inverse scope); `that`-scope ends at simple sentence unless `that` repeated; plural default collective, `each of` = distributive; countable beats mass; ditransitive beats transitive+PP; anaphora: proper names always accessible, indefinites inaccessible from under negation/modal/subordination/question/command (DRT accessibility), if-part accessible in then-part, disjuncts only forward; pronoun → most recent accessible agreeing non-subject NP; definite NP → most recent+most specific antecedent **else silently becomes new indefinite** (major failure mode); unknown capitalized word → guessed proper name; reflexive → sentence subject.
- Bidirectional: yes at DRS level (see APE ¶). PENS class (Kuhn 2014): **P4 E3 N4 S3, F W A** — deterministic parse, FOL-level expressiveness (rules+negation, no 2nd-order), natural sentences but clumsy texts, exact description >10 pages.
- Papers: Fuchs & Schwertel & Schwitter, ACE original (arXiv cmp-lg/9603003, 1996/1999); Fuchs, Kaljurand, Kuhn "ACE for Knowledge Representation", Reasoning Web 2008, doi:10.1007/978-3-540-85658-0_3; ACE 5 overview FLAIRS 2006.
- URLs: http://attempto.ifi.uzh.ch/site/docs/ace/6.7/ace_constructionrules.html · …/ace_interpretationrules.html · http://attempto.ifi.uzh.ch/site/pubs/papers/drs_report_66.pdf · https://en.wikipedia.org/wiki/Attempto_Controlled_English

### APE (ACE Parsing Engine)
- Reference parser, SWI-Prolog DCG. Deterministic: ACE text → exactly one DRS. Version "6.7-131003" (Oct 2013) but repo alive: last commits **2024-04** (httpserver tweaks). **License LGPL-3.0** (LICENSE.txt; GitHub shows NOASSERTION — file check confirms LGPL).
- Outputs (= compile targets): DRS (Prolog term/XML/pretty), **paraphrases** (`-cparaphrase1` Core ACE full-sentence, `-cparaphrase2` NP-ACE relative-clause), OWL 2/SWRL (FSS/XML/RDF), RuleML, FOL + prenex NF, TPTP. Modules: `prolog/parser/ace_to_drs.pl`, `prolog/utils/drs_to_ace.pl` (**DRACE = DRS→ACE verbalizer**), `drs_to_coreace.pl`, `drs_to_npace.pl`, `drs_to_fol_to_prenex.pl`, `drs_to_tptp.pl`, `owlswrl/`; `tests/test_drace.pl`, `examples/paraphrase_roundtrip.pl`.
- **Bidirectionality: real.** ace_to_drs + drs_to_ace gives parse ∘ verbalize round-trip; paraphrase = verbalize(parse(text)) into canonical "Core ACE". Caveat: DRACE covers DRSs of the shape APE emits; arbitrary hand-built DRSs outside that fragment fail/partial (UNCERTAIN exact coverage bound — undocumented).
- Lexicon: function words compiled in; content lexicon pluggable (`-ulexfile`), small default `prolog/lexicon/clex_lexicon.pl` ships under APE's LGPL.
- URL: https://github.com/Attempto/APE (220★). Web demo + webclient repo https://github.com/Attempto/cape (JS, pushed 2025-04, no license file).

### Clex (big lexicon)
- ~100,000-entry English lexicon for APE (Prolog facts: noun/verb/adj forms + gender). **GPL-3.0** (stricter than APE!). Last push 2018. CKC needs a domain lexicon anyway → GPL Clex = dev-time only, never vendor. https://github.com/Attempto/Clex

### DRS→ACE / OWL→ACE verbalizers
- DRACE + Core-ACE/NP-ACE paraphrasers: inside APE (above), LGPL-3.0.
- **OWL verbalizer** (Kaljurand): standalone SWI-Prolog, OWL 2 axioms → ACE, deterministic template-per-axiom-type; used by AceWiki + ACE View (Protégé plugin). **LGPL-3.0**, last push 2023-10. https://github.com/Kaljurand/owl-verbalizer · doc: http://attempto.ifi.uzh.ch/site/docs/owl_to_ace.html
- Paper: Kaljurand & Fuchs, "Verbalizing OWL in ACE" (OWLED 2007).

### AceRules
- Rule engine over ACE: ACE text → rules → answers, **answers + proof traces verbalized back to ACE** (`Answertext` = verbalized answerset; `ACETrace` = verbalized courteous derivation trace). SWI-Prolog + Java webapp. **LGPL-3.0**, last push **2024-11** (dockerized). Accepts ACE subset mappable to rule shapes (authoring doc: "AceRules… accepts only ACE sentences that can be mapped to rule structures").
- **Semantics offered: `court` (courteous logic programs: priorities + strong negation + NAF, guaranteed single answer set), `stable` (ASP answer sets, 0..n answers), `stable_strong` (ASP + strong negation).** Well-founded: discussed in the papers as candidate, **not implemented**. NAF surface = `does not provably` / `it is not provable that`; strong negation = ordinary ACE negation.
- Follow-on: **"Defeasible AceRules: A Prototype"** (Diller, Wyner, Strass, IWCS 2017) — ACE → defeasible rules/argumentation. https://aclanthology.org/W17-6805/
- Paper: Kuhn, "AceRules: Executing Rules in Controlled Natural Language", RR 2007, doi:10.1007/978-3-540-72982-2_24.
- URLs: https://github.com/tkuhn/AceRules · http://attempto.ifi.uzh.ch/site/docs/acerules_webservice.html · demo http://attempto.ifi.uzh.ch/acerules/

### RACE reasoner
- FO reasoner natively on ACE/DRS (Prolog, Satchmo-style model generation + auxiliary axioms for plurals/numbers). Modes: `check_consistency` | `prove` | `answer_query`; returns **all minimal inconsistent axiom subsets / all minimal proofs** with axiom provenance — i.e. exactly the conflict-localization shape CKC gets from Z3 unsat cores. Some non-monotonic support (Fuchs, "Reasoning in ACE: Non-monotonicity", CNL 2016, doi:10.1007/978-3-319-41498-0_2; earlier "First-Order Reasoning for ACE", CNL 2010 proceedings LNCS 7175, chapter DOI UNCERTAIN).
- **Distribution: hosted SOAP webservice + web client only; source never released; no license** (UNCERTAIN whether obtainable on request). github.com/fnogatz/race = MIT SWI-Prolog *client* for the SOAP API, last push 2018, not the reasoner.
- URLs: http://attempto.ifi.uzh.ch/race/ · http://attempto.ifi.uzh.ch/site/docs/race_webservice.html · https://github.com/fnogatz/race

### ACE-in-GF
- Subset of ACE 6.7 **syntax** re-implemented in Grammatical Framework, ported to ~15-19 languages via GF RGL: active Makefile list = Bul Cat Dut Eng Est Fre Ger Gre Ita Pol Ron Rus Spa Swe Urd (+ disabled Fin Dan Lav Nor) + abstract "Ace/Ape" concretes. **Does NOT implement DRS mapping** — pure multilingual surface; semantics still requires APE on the English rendering. Supports GF's incremental parsing → lookahead editors (used in AceWiki-GF). No license file. Last push **2021-09**.
- **Japanese: absent from ACE-in-GF, but GF RGL has Japanese (Jpn)**: paradigms ++, lexicon +, Lang +, API +, tested +, published + (missing Symb/Irreg/big-Dict/WordNet rows) → writing a Japanese concrete syntax for an ACE-like abstract grammar is feasible; RGL porting cost per language historically days-weeks. This is the **only Japanese story in the whole Attempto family** — ACE itself is English-only by definition.
- Papers: Angelov & Ranta, "Implementing Controlled Languages in GF", CNL 2009, doi:10.1007/978-3-642-14418-9_6 (5 langs, effort study); Kaljurand & Kuhn, "A Multilingual Semantic Wiki Based on ACE and GF", ESWC 2013, doi:10.1007/978-3-642-38288-8_29.
- URLs: https://github.com/Attempto/ACE-in-GF · https://www.grammaticalframework.org/lib/doc/status.html

### AceWiki (+ predictive editor / Codeco)
- Semantic wiki: content = ACE subset, stored as OWL-compatible axioms; every fact entered via **predictive editor** — chart parser over the grammar computes the exact set of legal next tokens, UI offers only those → **unparseable input is unrepresentable** (the "lookahead approach"). Grammar written in **Codeco**: grammar notation designed for predictive editing — declarative forward/backward **anaphora reference markers** in rules so the editor can offer only resolvable pronouns/definite NPs; compiles to Prolog DCG + Java chart parser; exhaustive lookahead is decidable. Java, **LGPL-3.0+**, last push **2026-02** (Kuhn still tending it). AceWiki-GF variant = multilingual via ACE-in-GF.
- Papers: Kuhn, "AceWiki: A Natural and Expressive Semantic Wiki" (2008, arXiv:0807.4618); Kuhn, "A Principled Approach to Grammars for CNLs and Predictive Editors", J. Log. Lang. Inf. 2013, doi:10.1007/s10849-012-9167-z; "Codeco: A Practical Notation…", CNL 2010, doi:10.1007/978-3-642-31175-8_6.
- URLs: https://github.com/AceWiki/AceWiki · http://attempto.ifi.uzh.ch/acewiki/ · http://attempto.ifi.uzh.ch/site/docs/authoring_tools.html

### Kuhn 2014 survey + PENS (scheme CKC will reuse)
- Kuhn, "A Survey and Classification of Controlled Natural Languages", Computational Linguistics 40(1):121-170, 2014, doi:10.1162/COLI_a_00168 · https://aclanthology.org/J14-1005/ · arXiv:1507.01701. 100 English CNLs, 1930-2014.
- **PENS**: 4 orthogonal descriptive (non-evaluative) dimensions × 5 classes = 625 cells; English = P1E5N5S1, propositional logic = P5E1N1S5. **Precision** P1 imprecise … P2 less-imprecise … P3 reliably interpretable (heuristics/feedback needed) … **P4 deterministically interpretable (formal grammar, deterministic parse to formal rep, semantics may stay underspecified)** … P5 fixed semantics (entailment fully defined, no heuristics). **Expressiveness** via features (a) universal quantification (b) arity>1 relations (c) general rule structures (d) negation (strong or NAF) (e) 2nd-order: E1 lacks a/b (prop. logic) · E2 a+b (DLs) · E3 +c+d (FOL) · E4 +e · E5 anything humanly communicable. **Naturalness**: N1 unnatural/symbolic · N2 dominant unnatural · N3 natural elements dominate, not valid sentences · N4 valid natural sentences, texts clumsy · N5 natural texts/flow. **Simplicity** = length of an exact+comprehensive syntax+semantics description: S1 impossible (natural lang) · S2 impossible but heavily restricted · S3 >10 pages · S4 1-10 pages · S5 ≤1 page. Extra letter codes: goal C comprehensibility / T translation / F formal-representation; W written / S spoken; D domain-specific; origin A academia / I industry / G government. Observed correlation: precision↑↔simplicity↑ (ρ=0.90). ACE = P4E3N4S3 FWA; AceWiki-subset languages land P4/P5 with S4.
- CKC's IR-as-CNL would target **P5 E3 N4 S4, F W D** — PENS gives the vocabulary to say precisely that.

### Post-2015 ACE×LLM
- **No published system yet wires ACE's grammar into LLM constrained decoding** (searched 2023-2025; gap = CKC's opportunity). Adjacent evidence: Shin et al., "Constrained Language Models Yield Few-Shot Semantic Parsers" (EMNLP 2021, arXiv:2104.08768) — constrained decoding into a *canonical CNL* then deterministic map to logical form beats decoding logical forms directly = CKC's exact architecture thesis. Grammar-constrained decoding toolchain mature (Geng et al. 2023; Outlines/SynCode/XGrammar; ACE-in-GF's GF grammar is machine-consumable for this). Garzo & Palumbo, "Human-in-the-Loop: Legal Knowledge Formalization in ACE", ISDFS 2025, doi:10.1109/isdfs65363.2025.11011971 (https://hal.science/hal-05021540) — humans author legal ACE, APE validates; LLMs only discussed as contrast. Dávila, "Controlled Natural Language Models", CEUR-WS Vol-3799 (2024) — CNL+logic-programs+LLM positioning essay, ACE-adjacent (https://ceur-ws.org/Vol-3799/paper10PEG2.0.pdf).

## 2. Evidence for rules/regulations/clinical
- **Clinical practice guidelines, directly on point**: Shiffman, Michel, Krauthammer, Fuchs, Kaljurand, Kuhn, "Writing Clinical Practice Guidelines in Controlled Natural Language", CNL 2009 (LNCS 5972, 2010), doi:10.1007/978-3-642-14418-9_16 — rewrote ATS/CDC tuberculosis-control + otitis-media recommendation statements in ACE; found ACE handles condition+action recommendations but authors needed the modality (`should/must/may`) + fought vagueness/underspecification of source prose. Guideline-formalization pedigree = Shiffman (GEM/GLIA author).
- Rules/policies: ACE = designated CNL of EU FP6 **REWERSE** (Semantic-Web rules + policies working group; Reasoning Web 2008 chapter documents it). AceRules built for business-rule style KBs (RR 2007). Defeasible/legal rules: Diller et al. IWCS 2017; Garzo & Palumbo ISDFS 2025 (legal knowledge). Höfler, "Legislative Drafting Guidelines: How Different Are They from Controlled Language Rules…", CNL 2012, doi:10.1007/978-3-642-32612-7_10 (Zurich group, statute drafting ↔ CNL rules).
- Ontology/KR at scale: ACE↔OWL round-trip (Kaljurand thesis 2007; ACE View Protégé plugin; AceWiki user studies 2008-2013).

## 3. Design lessons for CKC
**Copy:**
- **Attachment/scoping by decree**: kill ambiguity with blunt positional rules, not smarter parsing — PP→verb always; relative clause→adjacent noun; scope = surface order to sentence end; repeat the subordinator (`that`, `who`) to extend scope; comma-variants (`,and`) as explicit precedence override. Cheap to state in a grammar, trivially deterministic, and (per constrained decoding) trivially enforceable on the model.
- **Hyphenation trick**: lexical gluing (`fills-in`, `fond-of`, `United-Nations`) turns multiword units into single tokens → removes whole ambiguity classes pre-parse. For CKC: hyphenate/quote clinical multiword concepts (`community-acquired-pneumonia`) instead of parsing noun compounds.
- **Modal quadruple as deontic surface**: `must/should/may/cannot` + sentence forms `it is recommended that …` map 1:1 onto direction×strength (require/strong-recommend/permit/against). And ACE's honesty — DRS `should/1`, `may/1` declared "no general semantics, domain interprets" — is the right architecture: keep deontic force as labeled wrapper nodes in the IR, define semantics only in the compiler targets (SMT conflict axioms; Prolog rule priorities).
- **Verbalize-the-canonical-form loop**: APE's paraphrase = verbalize(parse(x)) into Core ACE — auditors read back what the machine *understood*, not what was written. CKC: always render the audit CNL from the IR (one canonical verbalizer), never echo model output; round-trip test = parse(verbalize(IR)) == IR, property-testable.
- **AceRules' answer/trace verbalization**: verbalizing not just rules but *inference traces* back to CNL = clinician-auditable explanations for the expert system. Courteous semantics (priorities + strong negation + single guaranteed model) fits guideline exceptions ("unless contraindicated") better than raw ASP multi-models.
- **Codeco insight for constrained decoding**: lookahead needs anaphora accessibility encoded *in the grammar* (forward/backward reference markers), else the constraint offers pronouns/definite refs with no antecedent. If CKC's CNL has any `it/the patient` style references, put referent-tracking in the grammar the decoder is constrained by — or design refs out (explicit variables `X`), which ACE also supports and which is the safer choice for machine authors.
- **PENS as design dashboard**: state CKC-CNL's target cell (P5 E3 N4 S4 D) and defend each dimension; S4 (≤10-page exact spec) is a good forcing function for grammar compactness = constrained-decoding compactness.
- **Layered lexicon**: closed function-word core in the grammar + pluggable domain lexicon (APE `-ulexfile`) = exactly the split constrained decoding wants (static grammar automaton + dynamic terminal sets from the guideline's concept dictionary).

**Avoid:**
- **Silent definite-NP fallback**: unresolved `the X` becomes a *new indefinite* — silent meaning drift; a weak model will trip this constantly. CKC: unresolved reference = hard parse error, or forbid definites entirely (use variables).
- **Unknown-capitalized-word = proper-name guessing** (`Guess on`): same silent-acceptance disease. Fail closed on OOV.
- **Anaphora/accessibility machinery at all**: biggest source of ACE's spec complexity (S3) and of Codeco's existence; machine-written rules don't need pronouns. One-sentence-one-rule, explicit variables → whole chapters of the spec vanish.
- **Full-English verbosity**: ACE is token-expensive (`It is recommended that a patient who is older than 18 years and who has X receives Y.`) — fine as *rendered audit surface*, wasteful as *model-emitted* form. CKC already separates IR (emit) from CNL (render); keep ACE-style text on the render side only. (ACE never had this split; its parser-first design forced humans to write the verbose form.)
- **Semantics-free multilingualism**: ACE-in-GF renders 15+ languages but only English parses back to DRS → multilingual *display*, monolingual *truth*. For Japanese audit surface: acceptable if Japanese is verbalize-only, but say so explicitly; a JP round-trip needs the grammar (GF RGL Jpn exists) *plus* a JP-aware lexicon mapping, budget accordingly.
- **Monotonic-FOL-only core**: base ACE has no exceptions/defeasibility; it got bolted on three separate times (NAF `provably`, AceRules courteous, Defeasible AceRules 2017) — design exception/override semantics into the IR from day one (CKC already does).
- **No native numeric intervals or durations**: `age >= 18` is expressible (`… is older than 18 years`? actually via comparison formulas `X >= 18` or generalized quantifiers), but "7 to 14 days" has no range construct — you'd paraphrase (`at least 7 days and at most 14 days`). CKC's interval atoms need purpose-built sentence patterns; ACE offers none to copy.
- **Webservice-only reasoner (RACE)**: closed source, single hosted instance, SOAP — unusable as a durable oracle. Anything CKC depends on must be runnable locally (APE/AceRules qualify; RACE doesn't).

## 4. Fit verdicts
| System | Verdict | Reason |
|---|---|---|
| ACE language design | **adapt** (imitate, reimplement) | Interpretation-rule playbook + modal quadruple + measurement/comparison syntax = direct blueprint for CKC's audit CNL; language itself too big (S3), anaphora-heavy, no intervals/temporal, English-only. Imitating a language design has no license problem. |
| APE | **mine-ideas + dev-time oracle** | LGPL-3.0 OK to *run* as reference parser/paraphraser for differential testing; SWI-Prolog DCG unfit to embed in CKC's deterministic Rust-ish pipeline; DRACE proves DRS-level round-trip is practical. |
| DRACE/Core-ACE/NP-ACE verbalizers | **adapt** | Canonical-paraphrase-from-semantics pattern = CKC's IR→CNL renderer; reimplement over CKC IR (tiny — template-per-node). |
| OWL verbalizer | mine-ideas | Deterministic template-per-axiom verbalization style transfers; OWL target itself irrelevant to CKC. |
| AceRules | **adapt** | Courteous semantics (priorities, strong-neg, one model) = right executable-rule-base semantics for guideline exceptions; ACE→rules→ACE-trace loop = the auditability story. LGPL, runnable oracle; reimplement semantics clean-room in CKC's Prolog-family target. |
| RACE | **reject** (mine the interface idea) | Closed-source hosted SOAP only; its "all minimal inconsistent axiom sets" output shape is worth copying — CKC already has it via Z3 unsat cores. |
| ACE-in-GF | mine-ideas | Proves grammar-portable multilingual rendering; no Japanese concrete, no semantics, unlicensed, dormant since 2021. If CKC wants GF for JP verbalization, start from GF RGL Jpn directly, not this repo. |
| Clex | reject | GPL-3 + general-English; CKC needs a clinical domain lexicon anyway. |
| AceWiki | mine-ideas | Predictive-editor/lookahead = the pre-LLM ancestor of grammar-constrained decoding; Codeco's referent-aware grammar constraints inform CKC's decoder grammar. Wiki itself irrelevant. |
| PENS scheme | **adopt** | Descriptive vocabulary for CKC IR/CNL design docs; zero implementation cost. |
